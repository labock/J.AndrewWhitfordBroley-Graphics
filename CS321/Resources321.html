RESOURCES—Useful Materials

To run WebGL programs, all you need is a computing device with an OpenGL-ready display and a web browser that is HTML 5.0 and WebGL compliant. Personal computers running Linux (natively), MacOSX, and Windows 7.0 and higher are generally equipped for WebGL; personal devices running iOS will probably work, but Android devices will probably not; the Linux virtual machine we use on campus does not support WebGL because it doesn't have access to the GPU. Most modern browsers, including Chrome, Firefox, and Safari, will support WebGL; Internet Explorer, unsurprisingly, does not, but the new Microsoft browser in Windows 10 probably does.

To develop WebGL programs, you need a text editor, preferably one that supports HTML and JavaScript syntax. I recommend the Bluefish editor for HTML files; it also supports JavaScript files, and it's available for all the major operating systems. On Linux, the gedit editor is a good alternative; in a pinch, you can use WordPad but not NotePad on Windows. If you use the CS Department computers, I recommend that you do your editing in the Linux virtual machine, but you'll need to display your programs using Chrome on the Windows side.

Guidelines for developing WebGL programs.

Here are the steps to develop and run WebGL programs:

  1. Always start with an existing program that already loads and runs. Use a program that is as similar as possible to the one you want to develop. To avoid plagiarism, include comments to identify the source you are building on, as well as comments to identify yourself, your collaborators (if any), and your contribution to the project.
  2. Both textbook and course sample programs assume that you have the textbook Common folder in the parent directory of the one you are developing the program in, so make your own copy of that folder and put it in the directory where your programs will go. In general, put each program in its own directory, with that directory in the same directory as the Common folder.
    Please do not put these programs in publicly accessible places (i.e., on a public web page) until after the course is over. As usual, you have a responsibility to protect your class files from access by other students during this time.
  3. Copy the source page (.html) file and its associated JavaScript (.js) file and rename them appropriately, if necessary.
  4. To view the program, right-click on the page file icon in a directory window, and choose “Open-with” and the Chrome browser or another WebGL-enabled broswer.
  5. To assist with debugging, you can open the JavaScript console in your browser. Errors in loading and running JavaScript display there. To open the console in Chrome on Windows, select Chrome Menu > More Tools > Developer tools, and then click the Console tab. (The Chrome menu is the thingy with three bars in the upper right corner.)

Tutorials

There are online tutorials for everything you need for this course. The website w3schools.com has excellent tutorials for both HTML and JavaScript. You won't necessarily need to learn too much more HTML than the sample programs provide, but you'll probably want some extra help with JavaScript.

Between the textbook and the class, we will cover pretty much all the WebGL you need to know for the course. At the other end of things, the WebGL Programming Guide listed on the course homepage tells you everything you could possibly want to know, if not always in the easiest to understand format. There are many tutorials online. One tutorial that takes a step-by-step interactive learning approach is WebGL Academy.
